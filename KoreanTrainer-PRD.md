> 基于 Nihondex 体系的韩语字符与语言训练平台
> 作者：Raymone | 版本：v1.0 | 日期：2026-04-05

## 1. 产品概述

### 1.1 定位

一个参照 Nihondex（日语学习平台）体系架构的韩语训练工具。核心理念：Nihondex 已经验证了「SRS + 游戏化 + 结构化分级」这套框架的有效性，本项目将该框架移植到韩语领域，填补市场空白。

### 1.2 目标用户

首要用户为开发者本人（Raymone），用于个人韩语学习。架构预留多语言扩展能力，后续可复用同一套系统扩展英语 IPA 音标模块。

### 1.3 技术栈（推荐）

- 前端：Next.js + React + Tailwind CSS
- 后端/数据库：Supabase（PostgreSQL + Auth + Realtime）
- 部署：Vercel
- 音频：Web Audio API + 预录音频文件
- PWA 支持：Service Worker + manifest.json（离线能力）

### 1.4 开发阶段

| 阶段 | 内容 | 优先级 |
|------|------|--------|
| v1 | 字符训练（核心刷题） | P0 |
| v2 | SRS 间隔重复系统 | P1 |
| v3 | 游戏化系统（岛屿/成就） | P2 |
| v4 | 进度追踪和统计 | P3 |

### 2.3 音节块系统（韩语特有，日语没有的维度）

韩语独特之处：单个字母组合成「音节块」。这是 Nihondex 没有的训练维度，也是 HangulDex 的差异化功能。

```
音节块结构：SyllableBlock {
  id: string
  block: string           // 完整音节，如 "한"
  initial: string         // 初声（辅音），如 "ㅎ"
  medial: string          // 中声（元音），如 "ㅏ"
  final: string | null    // 终声（收音/받침），如 "ㄴ"，可为空
  romanization: string    // "han"
  audio_url: string
  structure: enum         // "CV" | "CVC" — 有无收音
}
```

音节块组合规则：
- 类型 1（CV）：辅音 + 元音 → 如 가 (g + a)
- 类型 2（CVC）：辅音 + 元音 + 收音 → 如 한 (h + a + n)
- 收音（받침）共 27 种组合，但实际发音只有 7 种代表音：[ㄱ, ㄴ, ㄷ, ㄹ, ㅁ, ㅂ, ㅇ]

## 3. 阶段一：字符训练（v1 — P0 核心）

> 目标：完成韩文字母（자모）+ 音节块的识别与刷题功能，可独立使用。

### 3.1 功能模块

#### 3.1.1 字母学习（Learn 模式）

对标 Nihondex 的 Kana 学习页面。

展示界面：
- 字母表总览页（类似五十音图的网格展示）
- 按分组展示：基础辅音 | 基础元音 | 双辅音 | 复合元音
- 每个字母的详情卡片包含：字符、罗马音、IPA、发音音频播放、示例音节、示例词汇
- 点击字符自动播放发音

分组逻辑（辅音按发音位置分组，对应 Nihondex 的行分组）：
- 双唇音组：ㅁ, ㅂ, ㅃ, ㅍ
- 齿龈音组：ㄴ, ㄷ, ㄸ, ㅌ, ㄹ
- 齿龈擦音组：ㅅ, ㅆ
- 龈腭音组：ㅈ, ㅉ, ㅊ
- 软腭音组：ㄱ, ㄲ, ㅋ
- 喉音/零声母组：ㅇ, ㅎ

元音按阴阳性分组：
- 阳性元音：ㅏ, ㅑ, ㅗ, ㅛ（笔画朝右/上）
- 阴性元音：ㅓ, ㅕ, ㅜ, ㅠ（笔画朝左/下）
- 中性元音：ㅡ, ㅣ

#### 3.1.2 刷题模式（Quiz/Drill 模式）

对标 Nihondex 的 Kana Practice。核心交互循环：看到字符 → 选择/输入对应罗马音 → 反馈对错 → 下一题。

题目类型：

**Type A — 字符识别（看字选音）**
- 屏幕显示一个韩文字母（如 ㄱ）
- 4 个罗马音选项供选择（如 g, n, d, m）
- 选对 → 绿色反馈 + 播放正确发音
- 选错 → 红色反馈 + 显示正确答案 + 播放发音

**Type B — 听音识字（听声选字）**
- 自动播放一个字母的发音
- 4 个字符选项供选择
- Nihondex 没有这个模式，这是 HangulDex 的增强

**Type C — 罗马音识字（看音选字）**
- 屏幕显示罗马音（如 "g/k"）
- 4 个韩文字符选项
- 反向训练，强化双向记忆

**Type D — 音节块组装（韩语特有）**
- 屏幕显示目标音节块（如 "한"）
- 用户从字母池中选择正确的初声 + 中声 + 终声组合
- 或反向：给出字母组合，选择正确的音节块
- 这是拼图式交互，Nihondex 完全没有的游戏类型

**Type E — 收音识别（받침 训练）**
- 屏幕显示含收音的音节（如 "각"）
- 用户选择该收音的实际发音代表音（ㄱ → [k]）
- 针对 27 种收音到 7 种代表音的映射规则

刷题设置（用户可配置）：
- 字符范围选择（勾选要练的分组）
- 题目类型选择（A/B/C/D/E 任意组合）
- 每轮题数：10 / 20 / 50 / 无限
- 是否计时
- 是否显示提示（如发音位置图）

#### 3.1.3 易混淆字符专练

对标 Nihondex 的 Kanji Imposters 小游戏。

韩语中的高频混淆对：
- 视觉相似：ㅏ vs ㅓ, ㅗ vs ㅜ, ㄱ vs ㅋ, ㄴ vs ㄹ
- 发音相似：ㅐ vs ㅔ（现代韩语中几乎合并）, ㅚ vs ㅞ vs ㅙ
- 松音/紧音/送气音三组：ㄱ/ㄲ/ㅋ, ㄷ/ㄸ/ㅌ, ㅂ/ㅃ/ㅍ, ㅅ/ㅆ, ㅈ/ㅉ/ㅊ

专练模式：
- 每次只聚焦 2-3 个易混淆字符
- 快速轮替出题，强制辨别
- 错误率高的对子自动提高出题权重

## 4. 阶段二：SRS 间隔重复（v2 — P1）

> 目标：在 v1 刷题基础上叠加科学复习调度，让记忆从短期转为长期。

### 4.1 算法：SM-2（SuperMemo 2）

对标 Nihondex 的 SRS 系统。每个字符/音节是一个独立的 SRS 卡片。

```
SRS 卡片数据结构：SRSCard {
  item_id: string          // 对应 CharacterData.id 或 SyllableBlock.id
  ease_factor: number      // 难度因子，初始 2.5
  interval: number         // 当前间隔天数
  repetitions: number      // 连续正确次数
  next_review: Date        // 下次复习日期
  last_review: Date        // 上次复习日期
  total_reviews: number    // 总复习次数
  total_correct: number    // 正确次数
  status: enum             // "new" | "learning" | "review" | "known"
}
```

SM-2 核心逻辑：
- 用户评分 0-5（0-2 错误需重学，3 勉强记得，4 正确但犹豫，5 秒答）
- 简化为刷题中的二元判定：答对 = 4分，答错 = 1分
- 答对：interval × ease_factor，ease_factor 微调
- 答错：interval 重置为 1，ease_factor 下降（最低 1.3）

### 4.2 SRS 功能

**每日复习队列：**
- 登录后显示"今日待复习"数量
- 到期卡片按优先级排序：过期越久越优先
- 新卡片每日上限可配置（默认 20）

**学习会话流程（对标 Nihondex 的 Lesson + Review）：**
1. Lesson：学习新卡片（展示字符详情 → 确认记住 → 进入简单测试）
2. Review：复习到期卡片（直接进入刷题 → 根据表现调整间隔）

**Leech 检测：**
- 连续答错 4+ 次的卡片标记为 leech（水蛭）
- 单独提示用户重点关注
- 对标 Nihondex 的 leech 系统

### 4.3 数据持久化

v2 阶段需要将 localStorage 升级为 Supabase：
- 用户认证（邮箱/Google OAuth）
- SRS 数据云端同步
- 跨设备访问

## 5. 阶段三：游戏化系统（v3 — P2）

> 目标：用 Nihondex 式的岛屿建设和成就系统，为枯燥的重复练习提供外在动机。

### 5.1 岛屿系统（对标 Nihondex 的 Floating Island）

**核心机制：**
- 用户拥有一座可自定义的浮空岛屿
- 完成学习任务获得「经验值」和「金币」
- 金币用于购买岛屿装饰物（建筑、植物、角色等）
- 岛屿随学习进度视觉成长

**韩国主题化：**
- Nihondex 用日式浮空岛，HangulDex 可用韩国文化元素
- 装饰物：韩屋（한옥）、泡菜坛子、石锅、樱花树、景福宫模型等
- 岛屿分区对应学习阶段（如：入门区 → 初级村 → 中级城 → 高级宫殿）

**经验值获取规则：**
- 完成一次 Lesson（学新卡片）：+10 XP
- 完成一次 Review（复习）：+5 XP
- 刷题模式每答对一题：+2 XP
- 连续 7 天登录：+50 XP bonus
- 完成某一分组全部字母学习：+100 XP

### 5.2 成就系统

对标 Nihondex 的 Achievements。

示例成就（分类）：

**里程碑类：**
- 「첫걸음」（第一步）— 完成第一个字符学习
- 「자음 마스터」（辅音大师）— 学完所有 14 个基础辅音
- 「모음 마스터」（元音大师）— 学完所有 10 个基础元音
- 「한글 완성」（韩文完成）— 学完全部 40 个字母
- 「음절 건축가」（音节建筑师）— 正确组装 100 个音节块

**连续类：**
- 「3일 연속」— 连续 3 天学习
- 「7일 연속」— 连续 7 天学习
- 「30일 연속」— 连续 30 天学习

**精度类：**
- 「완벽한 라운드」— 单轮刷题 100% 正确
- 「속사포」— 10 秒内连续答对 5 题

**SRS 类：**
- 「기억의 달인」（记忆达人）— 10 个字符达到 "known" 状态

## 7. 扩展预留：英语 IPA 模块

当韩语模块完成后，用同一架构扩展英语 IPA 音标训练。

### 7.1 数据集映射

| HangulDex 结构 | 英语 IPA 对应 |
|---|---|
| 基础辅音 14 | IPA 辅音 ~24 个（/p/, /b/, /t/, /d/...） |
| 基础元音 10 | IPA 元音 ~12 个（/iː/, /ɪ/, /e/, /æ/...） |
| 双辅音 5 | 无直接对应，可映射为辅音丛（clusters） |
| 复合元音 11 | 双元音 8 个（/aɪ/, /aʊ/, /eɪ/...） |
| 音节块组装 | 单词音标拼读（IPA → 单词） |
| 收音规则 | 连读规则（liaison/linking） |
| TOPIK 分级 | CEFR A1→C2 或自定义难度分级 |

### 7.2 技术架构要求

为了支持多语言扩展，v1 就需要在代码层面做好抽象：

```
/data
  /korean     → 韩语数据集
  /english    → 英语 IPA 数据集（后续添加）
/components
  /shared     → 通用 UI 组件（刷题卡片、选项按钮、音频播放器等）
  /korean     → 韩语特有组件（音节块组装器等）
  /english    → 英语特有组件（后续添加）
/lib
  /srs        → SRS 算法（语言无关）
  /quiz       → 刷题逻辑引擎（语言无关）
  /gamification → 游戏化系统（语言无关）
```

核心原则：**数据和逻辑分离。** 刷题引擎、SRS 算法、游戏化系统都是语言无关的通用模块，韩语和英语只是不同的数据集插入同一个引擎。
